    Depuis maintenant quelques années, la séparation du contenu et de la mise en forme du document est devenu une règle dans la création de pages webs. C’est pour cela qu’à partir d'aujourd'hui, les mises en formes de type CSS vont se trouver dans un où plusieurs fichiers CSS (extension .css) et seront appelés par la page HTML correspondante avec l'instruction suivante qui sera à placer dans le <head>

    <link rel=''stylesheet'' type=''text/css'' href=''style.css''>

    Le paramètre de type est facultatif en HTML5 De la même façon que l'on va pouvoir utiliser plusieurs feuilles de style dans un même fichier HTML, il sera possible d'utiliser plusieurs fois la même feuille de style dans plusieurs fichiers HTML, d’où un gain de productivité une fois ses feuilles de style préparées.

    Dans une feuille style, la format adapté est très simple et bien défini, il prend toujours la forme :

    Sélecteur { propriété : valeur ; }

  4. Héritage


    Les feuilles de style sont en cascade, comme le dit leur nom (Cascading Style Sheet) ce qui implique un héritage des propriétés.

    Un exemple explique mieux que les mots donc pour une page codées comme cela :

    <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN"
    "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
    <html>
    <head>
    <title>Héritage</title>
    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" />
    <link rel="stylesheet" type="text/css" href="style.css" />
    </head>
    <body>
    <h1>L'hériage</h1>
    <p>
    L'héritage fait que ce texte est en italic et en bleu.
    </p>
    </body>
    </html>    		 Fichier style.css

    body {
    color: #FF0000;
    font-style: italic;
    }
    p {
    color: #0000FF;
    }


    Résultat :
    Pour le texte, cela se voit très rapidement, mais en ce qui concerne la mise en page, l'héritage peut être beaucoup plus problématique, comme par exemple avec les marges, le padding ou encore les bordures qui peuvent être invisibles et donc pas forcément réparties comme voulue.

    2. ID et class

      A ces deux balises, on va chercher à rajouter un style particulier, qui ne correspond pas au style de base, puisque l'on ne peut pas modifier toutes les balises de type span ou de type div dans notre fichier css sans quoi on n’obtiendra pas la flexibilité désirée. C'est pour cela que l'on va créer des sélecteurs personnalisés que l'on utilisera comme bon nous semble.
      Les Ids sont utilisés pour désigner des éléments uniques dans le document, et concernent donc bien souvent un div et non pas un span qui lui va plutôt utiliser une classe. Les classes permettent au style d'être utilisé plusieurs fois et vont donc souvent servir pour la mise en forme de certains morceaux de texte dans la page, et cela de façon répétée, comme par exemple changer la couleur d'un groupe de lien, ou encore celle d'une partie du texte.
